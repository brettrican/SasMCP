## v1.3.0 ‚Äî Kali-style In-Server Version Updater (DRAFT)

**Status:** in development. `__version__` is `1.3.0-dev`. Do not tag.

### What's new

The new `updater` tool group adds four MCP tools that mirror Kali's
`apt update` / `apt list --upgradable` / `apt show` / `apt upgrade` flow,
operating against this repo's GitHub Releases (or the gated
`sassyconsultingllc.com` endpoint when a license key is set).

| Tool | apt analogue | Returns |
|------|--------------|---------|
| `sassy_update_check` | `apt update` | current vs. latest tag, upgradable bool, release URL |
| `sassy_update_list` | `apt list --upgradable` | per-asset rows (name, size, download URL, content type, download count) |
| `sassy_update_changelog` | `apt show <pkg>` | release notes body for any tag |
| `sassy_update_apply` | `apt upgrade` (download phase only) | downloads the chosen asset to `%LOCALAPPDATA%\SassyMCP\updates\` and prints the run command |

### Design notes

- **Apply does not execute the installer.** MSIs need UAC. The user runs
  the returned `msiexec /i ...` command when they're ready. This matches
  apt's contract: download is automatic, state-changing install is a
  deliberate command.
- **License-aware download path.** When `SASSYMCP_LICENSE_KEY` is set in
  the environment, `sassy_update_apply` rewrites the URL to
  `https://sassyconsultingllc.com/download/sassymcp/windows/<asset>?key=...`
  instead of the GitHub Releases URL.
- **5-minute cache** on `_refresh()` to avoid hammering the GitHub API
  on every check call. `force=True` bypasses it.
- **Version comparator** parses `1.3.0-dev`, `v1.2.0`, `V1.2.0`,
  `1.2.0-rc1` ‚Äî pre-release suffixes sort BELOW the matching release
  (so `1.3.0-dev < 1.3.0`).
