## v1.3.1 — Portable-first distribution

### What changed

Dev feedback was unanimous: nobody wanted an MSI installer for a tool they
intend to inspect, sandbox, or rip out. v1.3.1 drops the MSI as a release
asset and leads with the portable bundle.

- **No more MSI.** v1.3.0's `SassyMCP-v1.3.0.msi` was the last MSI cut.
  All future releases ship as zip + standalone exe only. `installer.wxs`
  and `build-msi.ps1` stay in the source tree for anyone who wants to
  bake their own MSI for Group Policy / Intune deployments — but no
  signed MSI is published.
- **Bundle renamed `*-portable.zip`** (was `*-bundle.zip`) so the name
  reflects what it is: extract anywhere, run a `.bat`, delete the folder
  to uninstall.
- **`install.ps1` / `uninstall.ps1` removed** from `deploy/` — they were
  registry-backup wrappers around `msiexec` and have no purpose without
  an MSI. The portable bundle has nothing to "uninstall."
- **README install section** rewritten portable-first. The license-gated
  one-line PowerShell installer is still there for users who want to
  pull just the standalone exe through the gated mirror.

### Why this is better for devs

- Zero registry footprint.
- No UAC prompt.
- No Add/Remove Programs entry to surprise anyone.
- Works from a USB drive, a network share, anywhere.
- Inspect the contents before running — every file is in the zip.
- Removal is `Remove-Item -Recurse <folder>`. There is no other state.

### What still works exactly the same

- All four updater tools from v1.3.0 — `sassy_update_check`,
  `sassy_update_list`, `sassy_update_changelog`, `sassy_update_apply` —
  now download the portable zip (or standalone exe) instead of an MSI,
  and `sassy_update_apply`'s `next_step` field returns an `Expand-Archive`
  command instead of an `msiexec /i` command.
- All shell-interceptor hardening from v1.3.0 (quoted-string strip,
  `allow_pattern` opt-in, `sassy_audit_false_positives`) is unchanged.
